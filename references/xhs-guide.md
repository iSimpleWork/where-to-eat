# 小红书口碑与避坑指南 (Xiaohongshu Research Guide)

本指南指导 Agent 如何利用小红书（Red Note）数据对大众点评初筛出的候选餐厅进行口碑校验、体验细化与避坑分析，确保推荐结果真实靠谱。

## 1. 搜索策略 (Search Strategies)

小红书的数据主要来自于真实用户的“笔记”（Notes），可以通过以下搜索关键词组合，挖掘出餐厅在口味、服务、环境、排队等方面的真实面貌。

### 常用检索 Query 模板
针对候选列表中的每家餐厅，在 `search_web` 中运行以下组合查询：

1. **综合口碑与氛围检索**：
   - 语法：`小红书 <店铺名称> <目的地> 怎么样` 或 `小红书 <店铺名称> 评价`
   - 示例：`小红书 荣叔黄鱼面 新天地 怎么样`
2. **避坑与踩雷检索**（核心，用于过滤网红店）：
   - 语法：`小红书 <店铺名称> 避坑` 或 `小红书 <店铺名称> 踩雷` 或 `小红书 <店铺名称> 难吃`
   - 示例：`小红书 荣叔黄鱼面 踩雷`
3. **排队与就餐建议检索**：
   - 语法：`小红书 <店铺名称> 排队` 或 `小红书 <店铺名称> 预约`
   - 示例：`小红书 荣叔黄鱼面 排队`

## 2. 深度口碑分析维度 (Sentiment Analysis)

在浏览搜索结果时，Agent 需重点研判并提取以下方面的信息：

### 口味与菜品（真好吃 vs 智商税）
- 是否存在“咸得发苦”、“分量极少”、“食材不新鲜”等负面口碑。
- 提取用户反复推荐的 **“必点菜”** 列表。
- 确认是否有被营销炒作但在小红书被密集吐槽的“雷区菜品”。

### 环境与氛围（实景 vs 照骗）
- 店内光线、桌椅间距、吵闹程度如何？
- 是否属于“滤镜网红店”（照片好看但现场破旧拥挤）。
- 景观位是否需要额外预订或设置最低消费。

### 服务与排队（现场体验）
- 正常用餐时段（午市/晚市）的平均排队时间（如“排队 2 小时起”、“基本不排队”）。
- 能否通过微信小程序或大众点评提前线上取号。
- 是否存在服务员态度恶劣、上菜极慢、强制消费等硬伤。

## 3. 最终筛选决策 (Final Decision Making)

结合小红书反馈，对初筛列表进行最终汰换：
- **一票否决**：若小红书上近期（近半月/一个月内）有大量关于“食物中毒/卫生极差”、“欺诈消费”、“态度极其恶劣”的负面笔记，即使点评评分极高，也应直接淘汰。
- **网红过滤**：点评评分高但小红书集中反馈“纯属照骗、极度难吃、性价比极低”的店铺予以过滤，更换为口碑更扎实的“老字号”或“宝藏社区店”。
- **精选推荐**：保留 3 家在口味、环境、服务综合体验上在小红书上受真实好评，且最契合用户就餐目的的餐厅。
